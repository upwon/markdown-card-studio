## 1. 项目目标

构建一个浏览器端优先的 Markdown 卡片编辑器：用户在中间编辑 Markdown，右侧实时看到自动分页后的社交媒体卡片，并能保存文档、切换主题与排版、导入/下载 Markdown，以及导出单页或全部 PNG。

首版以“小红书 3:4 卡片”作为核心场景，但底层尺寸与主题配置保持可扩展，不把产品锁死在单一平台。

## 3. 已确定的产品决策

### 3.1 强制分页语法

首版使用独立一行的专用注释：

```md
<!-- pagebreak -->
```

这样不会与 Markdown 水平分隔线 `---` 冲突。格式工具栏提供“分页”按钮自动插入该语法，帮助区明确说明用法。

### 3.2 本地优先

首版不需要账号、数据库或 API。文档与设置存入浏览器本地：

- `localStorage`：当前文档、主题、模板、样式、编辑器偏好
- IndexedDB：首版只预留接口；需要持久化的本地图片资源在 Phase 2 接入
- 自动保存采用防抖，并显示“保存中 / 已保存 / 保存失败”

### 3.3 图片策略

- 网络图片按 Markdown URL 渲染
- 本地图片首版支持选择或拖入并插入对象 URL
- 界面明确提示对象 URL 在刷新后可能失效
- 导出前检查图片是否成功加载；跨域图片污染画布时给出具体提示

### 3.4 导出策略

- 卡片 DOM 保持逻辑尺寸 900 × 1200，界面缩放仅作用于外层预览容器
- 导出时直接捕获未缩放的卡片节点，确保 PNG 为 900 × 1200
- 全部导出按页码命名，例如 `markdown-card-01.png`
- ZIP 文件包含全部 PNG，失败页面需要单独报告，不能静默跳过

## 4. 技术方案

### 4.1 基础技术栈

- Next.js 稳定版 App Router + TypeScript
- React 稳定版
- Tailwind CSS v4 + CSS variables 设计 token
- shadcn/ui 基础交互组件，Geist 作为应用界面字体
- CodeMirror 6：编辑、高亮、快捷键、历史、查找替换
- unified + remark-parse + remark-gfm：Markdown AST 管线
- Zustand：编辑器、文档、主题和预览状态
- DOM 图片导出库：在实现阶段用最小技术验证比较 `html-to-image` 与 `modern-screenshot`，以字体、代码块、跨域图片和像素精度为选择标准
- JSZip：批量 ZIP
- Vitest + Testing Library：单元与组件测试
- Playwright：桌面端、移动端和导出关键流程测试

依赖版本在编码开始时以官方最新稳定版为准，并确保 Next.js 与 React 使用已修复公开安全问题的版本，不采用 canary。

### 4.2 运行模式

编辑器主体是一个客户端交互应用。Next.js App Router 保留静态外壳和元数据，浏览器 API、CodeMirror、DOM 测量、本地存储与图片导出只在客户端边界内执行。

首版不引入 Server Actions、Route Handler 或服务端数据库，减少部署与隐私复杂度。

## 5. Markdown 与分页设计

### 5.1 内容管线

```text
Markdown 文本
  → 识别显式分页标记
  → remark 解析为带源码位置的 AST
  → 转换为内部块模型
  → 按显式分页切成内容段
  → DOM 测量驱动的自动分页
  → 页面模型
  → 卡片渲染
```

内部块模型保留：块类型、内容、AST 节点、源码起止位置、是否可拆分、是否应与下一块绑定等信息，为后续双向定位留下基础。

### 5.2 自动分页算法

1. 在不可见测量容器中使用与正式卡片完全一致的宽度、字体和样式。
2. 按块贪心装入当前页面，测量内容高度是否超过可用区域。
3. 标题与紧随其后的第一个正文块作为“尽量绑定”组合，避免标题落在页尾。
4. 图片、代码块、引用和表格默认是原子块；放不下时整体移到下一页。
5. 单个原子块超过一整页时显示明确的溢出状态；代码与表格允许在限定区域内滚动仅用于预览，但导出前提示用户调整字号或内容。
6. 普通段落和列表超长时，按文本/列表项进行二分切分，并保留 Markdown 语义。
7. 页面底部预留安全区，避免页脚与正文碰撞。
8. 等待字体和图片稳定后重新测量；使用 `ResizeObserver` 处理窗口和设置变化。

### 5.3 防止不稳定

- 解析与分页以递增任务 ID 管理，丢弃过期结果
- 对字体、图片和容器尺寸变化进行批量重排，避免重复抖动
- 每页保存稳定的块 key，减少 React 重建
- 设置最大重排次数并在异常时显示诊断提示
- 测试覆盖空文档、单个超高块、连续标题、长列表、长代码、宽表格、图片加载失败和连续分页标记

## 6. 主题与排版模板

主题只负责颜色，排版模板只负责结构和尺寸，用户覆盖值独立于二者。

主题 token 至少包含：

```text
paper, surface, ink, text, muted, accent, accentContrast,
line, quoteBackground, codeBackground, codeText, shadow
```

排版模板至少包含：

- `Deep Reading`：高密度、明确层级、适合知识型长文
- `Warm Paper`：更大留白、暖纸背景、适合舒适阅读

模板配置包含画布内边距、内容区间距、标题比例、正文字号、行高、圆角、装饰与页脚布局。组件不直接写死品牌颜色。

## 7. 界面与交互

### 7.1 桌面端

- 顶栏：新建、导入、下载 Markdown、撤销/重做、尺寸、缩放、导出
- 左栏：文档、模板、主题、画布、文字、间距、页脚设置
- 中栏：格式工具栏 + CodeMirror 编辑器 + 字数/保存状态
- 右栏：纵向卡片列表 + 页码 + 当前页选择 + 预览缩放

### 7.2 移动端

- 顶部精简操作栏
- 主区域使用“编辑 / 预览 / 样式”标签
- 底部固定当前页/全部导出入口
- 切换标签时保持编辑器选择区和预览滚动位置

### 7.3 初始体验

- 第一次打开加载一份覆盖主要语法与分页标记的中文示例文档
- 空文档显示简短引导，不生成无意义空卡片
- 导出、导入、自动保存均提供成功或失败反馈
- 本地数据覆盖或新建文档前提供确认，避免误删内容

## 10. 风险与应对

### DOM 分页结果受字体和图片加载影响

等字体与图片就绪后测量，监听尺寸变化并丢弃过期分页任务；准备固定 fixture 做重复稳定性测试。

### 超高代码、表格或图片无法优雅拆分

首版采用“原子块 + 明确溢出提示”，同时提供字号、边距调整；不做不可预测的静默裁剪。

### DOM 转图片的字体、滤镜或跨域图片兼容性

在 M5 先做最小技术验证；选择导出库前测试中英文字体、代码块、阴影、深色主题和跨域图片。导出失败必须给出可行动的原因。

### localStorage 容量和 schema 变化

持久化数据带版本号和迁移函数；首版不把大体积图片写入 localStorage。

### 客户端主包过大

CodeMirror、导出与 ZIP 模块按需动态加载；设置性能预算并在生产构建检查 bundle。
